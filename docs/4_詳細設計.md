## バックエンド実装詳細

### API エンドポイント設計
1. **ユーザー管理API**
   - POST /api/auth/register
     - リクエスト: `{ email, password, name, dateOfBirth }`
     - レスポンス: `{ user, token }`
     - 機能: ユーザー登録、JWT発行
   - POST /api/auth/login
     - リクエスト: `{ email, password }`
     - レスポンス: `{ user, token }`
     - 機能: 認証、JWT発行
   - GET /api/auth/me
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ user }`
     - 機能: 現在のユーザー情報取得
   - PUT /api/auth/profile
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ displayName, bio, preferences, avatar }`
     - レスポンス: `{ user }`
     - 機能: プロフィール更新

2. **レシピ管理API**
   - GET /api/recipes
     - クエリパラメータ: `search, category, ingredients, strength, sort, page, limit`
     - レスポンス: `{ recipes: [], total, page, pageSize }`
     - 機能: レシピ検索・一覧取得
   - GET /api/recipes/:id
     - レスポンス: `{ recipe, related, variants }`
     - 機能: レシピ詳細取得、閲覧カウント増加
   - POST /api/recipes
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ title, description, ingredients, steps, images, tags, strength, parentRecipeId? }`
     - レスポンス: `{ recipe }`
     - 機能: レシピ作成
   - PUT /api/recipes/:id
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ title, description, ingredients, steps, images, tags, strength }`
     - レスポンス: `{ recipe }`
     - 機能: レシピ更新（作成者のみ）
   - DELETE /api/recipes/:id
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ success: true }`
     - 機能: レシピ削除（作成者のみ）
   - POST /api/recipes/:id/like
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ success: true, likes }`
     - 機能: 「おいしい」追加
   - DELETE /api/recipes/:id/like
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ success: true, likes }`
     - 機能: 「おいしい」削除

3. **材料管理API**
   - GET /api/ingredients
     - クエリパラメータ: `category, search`
     - レスポンス: `{ ingredients: [] }`
     - 機能: 材料一覧取得
   - GET /api/users/me/ingredients
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ ingredients: [] }`
     - 機能: ユーザーの手持ち材料取得
   - POST /api/users/me/ingredients
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ ingredientId, quantity }`
     - レスポンス: `{ success: true }`
     - 機能: 手持ち材料追加
   - PUT /api/users/me/ingredients/:id
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ quantity }`
     - レスポンス: `{ success: true }`
     - 機能: 手持ち材料更新
   - DELETE /api/users/me/ingredients/:id
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ success: true }`
     - 機能: 手持ち材料削除

4. **通知API**
   - GET /api/notifications
     - ヘッダー: `Authorization: Bearer <token>`
     - クエリパラメータ: `page, limit, isRead`
     - レスポンス: `{ notifications: [], total, unreadCount }`
     - 機能: 通知一覧取得
   - PUT /api/notifications/:id/read
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ success: true }`
     - 機能: 通知を既読にする
   - PUT /api/notifications/read-all
     - ヘッダー: `Authorization: Bearer <token>`
     - レスポンス: `{ success: true }`
     - 機能: 全通知を既読にする

5. **AI API**
   - POST /api/ai/generate-recipe
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ preferences, ingredients, strength, mood }`
     - レスポンス: `{ recipe }`
     - 機能: AIカクテルレシピ生成
   - POST /api/ai/save-recipe
     - ヘッダー: `Authorization: Bearer <token>`
     - リクエスト: `{ recipe, customizations }`
     - レスポンス: `{ savedRecipe }`
     - 機能: 生成レシピ保存

### データベーススキーマ詳細
```sql
-- ユーザーテーブル
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  display_name VARCHAR(100) NOT NULL,
  date_of_birth DATE NOT NULL,
  role VARCHAR(20) NOT NULL DEFAULT 'user',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- ユーザープロフィールテーブル
CREATE TABLE user_profiles (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  bio TEXT,
  avatar_url VARCHAR(255),
  preferences JSONB,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 材料カテゴリテーブル
CREATE TABLE ingredient_categories (
  id UUID PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 材料マスターテーブル
CREATE TABLE ingredients (
  id UUID PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  category_id UUID REFERENCES ingredient_categories(id),
  alcohol_percentage FLOAT,
  image_url VARCHAR(255),
  affiliate_link VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- ユーザー所持材料テーブル
CREATE TABLE user_ingredients (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  ingredient_id UUID REFERENCES ingredients(id) ON DELETE CASCADE,
  quantity VARCHAR(20) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, ingredient_id)
);

-- レシピテーブル
CREATE TABLE recipes (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE SET NULL,
  title VARCHAR(255) NOT NULL,
  description TEXT NOT NULL,
  slug VARCHAR(255) UNIQUE NOT NULL,
  prep_time INTEGER,
  strength VARCHAR(20) NOT NULL,
  taste_profile JSONB,
  type VARCHAR(20) NOT NULL,
  parent_recipe_id UUID REFERENCES recipes(id),
  view_count INTEGER DEFAULT 0,
  featured BOOLEAN DEFAULT false,
  published_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- レシピ材料テーブル
CREATE TABLE recipe_ingredients (
  id UUID PRIMARY KEY,
  recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
  ingredient_id UUID REFERENCES ingredients(id),
  amount VARCHAR(50) NOT NULL,
  unit VARCHAR(20),
  note TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- レシピ手順テーブル
CREATE TABLE recipe_steps (
  id UUID PRIMARY KEY,
  recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
  step_number INTEGER NOT NULL,
  description TEXT NOT NULL,
  image_url VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- レシピ画像テーブル
CREATE TABLE recipe_images (
  id UUID PRIMARY KEY,
  recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
  image_url VARCHAR(255) NOT NULL,
  is_primary BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- レシピタグテーブル
CREATE TABLE recipe_tags (
  id UUID PRIMARY KEY,
  recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
  tag VARCHAR(50) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- いいねテーブル
CREATE TABLE likes (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, recipe_id)
);

-- フォローテーブル
CREATE TABLE follows (
  id UUID PRIMARY KEY,
  follower_id UUID REFERENCES users(id) ON DELETE CASCADE,
  following_id UUID REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(follower_id, following_id)
);

-- 通知テーブル
CREATE TABLE notifications (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  type VARCHAR(50) NOT NULL,
  source_id UUID,
  message TEXT,
  is_read BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 保存レシピテーブル
CREATE TABLE saved_recipes (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, recipe_id)
);
```

### 認証・認可フロー
- **JWT認証フロー**
  - トークン生成
    - ペイロード: `{ userId, email, role }`
    - 有効期限: 7日間
    - 署名アルゴリズム: HMAC SHA-256
  - トークン検証
    - ミドルウェアによる検証
    - 期限切れ処理
  - トークンリフレッシュ
    - 有効期限の70%経過時に自動リフレッシュ

- **ロールベースのアクセス制御**
  - ロール定義
    - `user`: 一般ユーザー
    - `admin`: 管理者
  - アクセス制御ミドルウェア
    - リソースオーナーシップ検証
    - ロール権限チェック

- **OAuth2.0連携**
  - Google OAuth実装
    - スコープ: `profile`, `email`
    - コールバックハンドリング
  - Facebook Login実装

### 入力検証
- **バリデーションルール詳細**
  - メールアドレス
    - 有効なメールアドレス形式
    - 最大長255文字
  - パスワード
    - 最小8文字
    - 英大文字・小文字・数字・特殊文字を含む
  - ユーザー名
    - 最小2文字、最大100文字
    - 英数字・記号一部のみ許可
  - レシピ関連
    - タイトル: 最小3文字、最大255文字
    - 説明: 最大10000文字
    - タグ: 最大10個、各タグ最大50文字

- **サニタイゼーション方法**
  - HTML特殊文字エスケープ
  - 入力トリミング
  - Markdownパーサによる安全な書式変換
  - アップロード画像のEXIFデータ削除

### 認証強化
- **多要素認証**
  - メールリンク確認
  - 将来的実装: TOTPベースの2FA

- **ブルートフォース対策**
  - レート制限
    - ログイン試行: 10回/10分
    - パスワードリセット: 3回/30分
    - 会員登録: 5回/60分
  - アカウントロックアウト
    - 連続5回失敗でロック（10分間）

- **セッション管理**
  - セッションタイムアウト（7日間）
  - デバイスフィンガープリント追跡
  - 異常ログイン検知
    - 新デバイス・新ロケーションでの通知

### データ保護
- **暗号化アルゴリズム選定**
  - パスワードハッシュ: bcrypt（コスト係数12）
  - JWTシグネチャ: HMAC SHA-256

- **PII（個人識別情報）保護対策**
  - 最小限の個人情報収集
  - 物理削除ポリシー（GDPR対応）
  - 安全でない保存なし（メモリキャッシュなど）

## 運用・保守計画

### バックアップ計画
- **バックアップスケジュール**
  - データベース完全バックアップ: 日次（UTC 02:00）
  - ユーザーコンテンツ増分バックアップ: 6時間ごと
  - 設定ファイルバックアップ: 変更時

- **リカバリ手順**
  - データベースリストア手順
    1. 最新バックアップからのリストア
    2. トランザクションログ適用
    3. 整合性検証
  - 部分リカバリ
    - 特定テーブルのみの復旧手順
    - ポイントインタイムリカバリ手順

### 更新計画
- **パッチ適用計画**
  - セキュリティパッチ: 公開後24時間以内
  - バグ修正: 週次リリース
  - 機能アップデート: 月次リリース

- **バージョン管理戦略**
  - セマンティックバージョニング（Major.Minor.Patch）
  - リリースノート作成ルール
  - 下位互換性維持ポリシー

### 障害対応計画
- **障害検知方法**
  - アラート閾値設定
    - エラーレート: 5%超過で通知
    - レスポンスタイム: p95 > 1000ms
    - サーバーステータス: 1分以上のダウンタイム
  - 合成モニタリング
    - 10分ごとの主要ページアクセス

- **エスカレーションフロー**
  - レベル1: 自動修復（キャッシュクリア、再起動）
  - レベル2: オンコール開発者通知
  - レベル3: チーム全体への緊急通知

- **ロールバック手順**
  - デプロイ前のスナップショット取得
  - 1クリックロールバックプロセス
  - ダウンタイムなしロールバック手順
