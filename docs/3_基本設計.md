# 基本設計

システム全体の構成と基本的な設計方針を記載する。

## 1. システム構成

### 1.1 アーキテクチャ概要

- JAMstack アーキテクチャ採用
  - フロントエンド：Next.js（SSRを検討）
  - バックエンド：Next.js API Routes
  - API連携：RESTful API
  - 状態管理：未定
  - UIコンポーネントライブラリ：Material-UI（MUI）
  - フォルダ構成：feature-basedを基本に、各feature内にtype-basedのハイブリッド構成

### 1.2 インフラストラクチャ

- クラウドプラットフォーム：Cloudflare
  - フロントエンドホスティング：Cloudflare Workers
  - データベース：Cloudflare D1（SQLite互換）
  - ストレージ：ひとまずCloudflare Static Asset（画像・静的アセット用）
  - CDN：Cloudflareエッジネットワーク

### 1.3 セキュリティ設計

- 認証・認可
  - JWTベースの認証システム
  - セッション管理（HTTPOnly Cookie）
  - CORS設定
  - レート制限（API呼び出し制限）
- データ保護
  - 個人情報の暗号化保存
  - HTTPS強制（TLS 1.3）
  - XSS/CSRF対策実装
- コンテンツセキュリティ
  - ユーザー生成コンテンツの検証
  - 画像アップロード制限（サイズ、形式）

## 3. データ設計

### 3.1 エンティティ関連図（ER図）

主要なエンティティとその関連性を定義：

- Cocktail（カクテル）
  - CocktailIngredient（カクテル材料）[1:n]
  - Instruction（手順）[1:n]
  - CocktailTag（カクテルタグ）[1:n]
- Ingredient（材料マスタ）
  - Category（材料カテゴリ）[n:1]
  - IngredientGroup（材料グループ）[n:1]
  - CocktailIngredient（カクテル材料）[n:1]
- Tag（タグマスタ）
  - CocktailTag（カクテルタグ）[n:1]

### 3.2 データモデル

各エンティティの属性とデータ型を定義

- `cocktails`
  - `id`: TEXT (主キー)
  - `name`: TEXT (NULL不可)
  - `slug`: TEXT (一意, NULL不可)
  - `description`: TEXT
  - `garnish`: TEXT
  - `imageUrl`: TEXT
  - `created_at`: TEXT (デフォルト CURRENT_TIMESTAMP)
  - `updated_at`: TEXT (デフォルト CURRENT_TIMESTAMP)
- `ingredients`
  - `id`: INTEGER (主キー, 自動インクリメント)
  - `name`: TEXT (一意, NULL不可)
  - `description`: TEXT
  - `group_id`: INTEGER (外部キー, `ingredient_groups`テーブルの`id`を参照, カスケード削除)
  - `category_id`: INTEGER (外部キー, `categories`テーブルの`id`を参照)
- `cocktail_ingredients`
  - `cocktail_id`: TEXT (複合主キー, 外部キー, `cocktails`テーブルの`id`を参照, カスケード削除)
  - `ingredient_id`: INTEGER (複合主キー, 外部キー, `ingredients`テーブルの`id`を参照, カスケード削除)
  - `amount`: TEXT (NULL不可)
  - `option_group`: INTEGER
- `instructions`
  - `id`: INTEGER (主キー, 自動インクリメント)
  - `cocktail_id`: TEXT (外部キー, `cocktails`テーブルの`id`を参照, カスケード削除)
  - `step`: INTEGER (NULL不可)
  - `text`: TEXT (NULL不可)
- `categories`
  - `id`: INTEGER (主キー, 自動インクリメント)
  - `name`: TEXT (一意, NULL不可)
  - `sortOrder`: INTEGER (NULL不可)
  - `icon`: TEXT
  - `description`: TEXT
- `ingredient_groups`
  - `id`: INTEGER (主キー, 自動インクリメント)
  - `displayName`: TEXT (一意, NULL不可)
  - `sortOrder`: INTEGER (NULL不可)
  - `description`: TEXT
  - `icon`: TEXT
- `tags`
  - `id`: INTEGER (主キー, 自動インクリメント)
  - `name`: TEXT (一意, NULL不可)
  - `description`: TEXT
- `cocktail_tags`
  - `cocktail_id`: TEXT (複合主キー, 外部キー, `cocktails`テーブルの`id`を参照, カスケード削除)
  - `tag_id`: INTEGER (複合主キー, 外部キー, `tags`テーブルの`id`を参照, カスケード削除)

### 3.3 API設計

RESTful API設計原則に従ったエンドポイント設計

- カクテル系
  - GET /api/cocktails - カクテル一覧取得
  - GET /api/cocktails/:slug - カクテル詳細取得
- 材料系
  - GET /api/ingredients - 材料一覧取得
- AI系
  - POST /api/generate-cocktail - AIカクテルレシピ生成
- 管理系
  - POST /api/admin/seed - 開発用データ投入

## 5. 非機能設計

### 5.1 性能設計

- CDN活用によるコンテンツ配信の高速化
  - 静的資産のエッジキャッシュ
  - キャッシュヘッダー最適化
- 画像最適化
  - WebP/AVIF形式対応
  - レスポンシブ画像配信
  - 遅延読み込み実装
- コード分割とレイジーローディング
  - ルートベースのコード分割
  - 動的インポート
- キャッシュ戦略
  - Service Worker実装（PWA対応）

### 5.2 可用性設計

- Cloudflareのグローバルネットワーク活用
  - エッジでのフォールバック設定
  - 自動復旧メカニズム
- ヘルスチェック
  - APIエンドポイント監視
  - Webhook通知設定
- エラーハンドリング
  - グレースフルデグラデーション
  - フォールバックUI
  - エラーバウンダリ実装
- バックアップ/リストア戦略
  - ポイントインタイムリカバリ
  - 障害復旧プラン

### 5.3 監視・運用設計

- モニタリング
  - アクセスログ収集・分析
  - エラー検知・通知
    - Sentry（エラートラッキング） を使用し、フロントエンドおよびバックエンドのエラーを捕捉し、エラーのグルーピングやアラート設定を行うことで、効率的なエラー管理を実現
  - パフォーマンス測定
    - Datadog を使用して、Web Vitals（LCP, FID, CLS, TTFB）やサーバーサイドメトリクス（レスポンスタイム、エラーレート、リクエスト数） の監視
    - 高エラーレート（5%超過）、高レイテンシ（p95 > 1000ms）、サーバーダウン、異常トラフィックなどの閾値を設定し、異常を検知した際にはアラートを通知
