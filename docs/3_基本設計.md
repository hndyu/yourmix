# 基本設計

システム全体の構成と基本的な設計方針を記載する。

## 1. システム構成

### 1.1 アーキテクチャ概要

- JAMstack アーキテクチャ採用  
  - フロントエンド：Next.js（SSRを検討）  
  - バックエンド：Next.js API Routes  
  - API連携：RESTful API  
  - 状態管理：未定  
  - UIコンポーネントライブラリ：Material-UI（MUI）
  - フォルダ構成：feature-basedを基本に、各feature内にtype-basedのハイブリッド構成

### 1.2 インフラストラクチャ

- クラウドプラットフォーム：Cloudflare  
  - フロントエンドホスティング：Cloudflare Workers  
  - データベース：Cloudflare D1（SQLite互換）  
  - ストレージ：Cloudflare R2を検討（画像・静的アセット用）  
  - CDN：Cloudflareエッジネットワーク

### 1.3 セキュリティ設計

- 認証・認可  
  - JWTベースの認証システム  
  - セッション管理（HTTPOnly Cookie）  
  - CORS設定  
  - レート制限（API呼び出し制限）  
- データ保護  
  - 個人情報の暗号化保存  
  - HTTPS強制（TLS 1.3）  
  - XSS/CSRF対策実装  
- コンテンツセキュリティ  
  - ユーザー生成コンテンツの検証  
  - 画像アップロード制限（サイズ、形式）

## 3. データ設計

### 3.1 エンティティ関連図（ER図）

主要なエンティティとその関連性を定義：

- User（ユーザー）  
  - UserProfile（ユーザープロフィール）[1:1]  
  - UserIngredient（ユーザー所持材料）[1:n]  
  - Recipe（レシピ）[1:n]  
  - Like（「おいしい」）[1:n]  
  - Follow（フォロー関係）[n:n]  
  - SavedRecipe（保存レシピ）[1:n]  
- Notification（通知）  
  - User（ユーザー）[n:1]  
- Recipe（レシピ）  
  - RecipeIngredient（レシピ材料）[1:n]  
  - RecipeImage（レシピ画像）[1:n]  
  - RecipeTag（レシピタグ）[n:n]  
  - SavedRecipe（保存レシピ）[n:1]  
  - RecipeStep（レシピ手順）[1:n]  
  - BrowsingHistory（閲覧履歴）[n:1]  
- Ingredient（材料マスタ）  
  - IngredientCategory（材料カテゴリ）[n:1]  
  - UserIngredient（ユーザー所持材料）[1:n]  
  - RecipeIngredient（レシピ材料）[n:1]

### 3.2 データモデル

各エンティティの属性とデータ型を定義

- `users`  
  - `id`: UUID (主キー)  
  - `email`: VARCHAR(255) (一意, NULL不可)  
  - `password_hash`: VARCHAR(255) (NULL不可)  
  - `display_name`: VARCHAR(100) (NULL不可)  
  - `date_of_birth`: DATE (NULL不可)  
  - `role`: VARCHAR(20) (NULL不可, デフォルト 'user')  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `user_profiles`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `bio`: TEXT  
  - `avatar_url`: VARCHAR(255)  
  - `preferences`: JSONB  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `ingredient_categories`  
  - `id`: UUID (主キー)  
  - `name`: VARCHAR(100) (NULL不可)  
  - `description`: TEXT  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `ingredients`  
  - `id`: UUID (主キー)  
  - `name`: VARCHAR(100) (NULL不可)  
  - `description`: TEXT  
  - `category_id`: UUID (外部キー、`ingredient_categories`テーブルの`id`を参照)  
  - `alcohol_percentage`: FLOAT  
  - `image_url`: VARCHAR(255)  
  - `affiliate_link`: VARCHAR(255)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `user_ingredients`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `ingredient_id`: UUID (外部キー、`ingredients`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `quantity`: VARCHAR(20) (NULL不可)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - UNIQUE(`user_id`, `ingredient_id`): 各ユーザーは同じ材料を重複して登録できません。  
- `recipes`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にNULLに設定)  
  - `title`: VARCHAR(255) (NULL不可)  
  - `description`: TEXT (NULL不可)  
  - `slug`: VARCHAR(255) (一意, NULL不可)  
  - `prep_time`: INTEGER  
  - `strength`: VARCHAR(20) (NULL不可)  
  - `taste_profile`: JSONB  
  - `type`: VARCHAR(20) (NULL不可)  
  - `parent_recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、派生レシピの場合)  
  - `view_count`: INTEGER (デフォルト 0)  
  - `featured`: BOOLEAN (デフォルト false)  
  - `published_at`: TIMESTAMP  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `recipe_ingredients`  
  - `id`: UUID (主キー)  
  - `recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、レシピ削除時にカスケード削除)  
  - `ingredient_id`: UUID (外部キー、`ingredients`テーブルの`id`を参照)  
  - `amount`: VARCHAR(50) (NULL不可)  
  - `unit`: VARCHAR(20)  
  - `note`: TEXT  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `recipe_steps`  
  - `id`: UUID (主キー)  
  - `recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、レシピ削除時にカスケード削除)  
  - `step_number`: INTEGER (NULL不可)  
  - `description`: TEXT (NULL不可)  
  - `image_url`: VARCHAR(255)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `recipe_images`  
  - `id`: UUID (主キー)  
  - `recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、レシピ削除時にカスケード削除)  
  - `image_url`: VARCHAR(255) (NULL不可)  
  - `is_primary`: BOOLEAN (デフォルト false)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - `updated_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `recipe_tags`  
  - id: UUID（主キー）  
  - recipe_id: UUID（外部キー）  
  - tag: VARCHAR(50)  
  - created_at: DATETIME  
  - updated_at: DATETIME  
- `likes`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、レシピ削除時にカスケード削除)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - UNIQUE(`user_id`, `recipe_id`): 各ユーザーは同じレシピに複数回「おいしい」を付けられません。  
- `follows`  
  - `id`: UUID (主キー)  
  - `follower_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `following_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - UNIQUE(`follower_id`, `following_id`): 同じフォロー関係は重複して登録されません。  
- `notifications`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `type`: VARCHAR(50) (NULL不可)  
  - `source_id`: UUID  
  - `message`: TEXT  
  - `is_read`: BOOLEAN (デフォルト false)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
- `saved_recipes`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、レシピ削除時にカスケード削除)  
  - `created_at`: TIMESTAMP (デフォルト CURRENT_TIMESTAMP)  
  - UNIQUE(`user_id`, `recipe_id`): 各ユーザーは同じレシピを複数回保存できません。  
- `browsing_histories`  
  - `id`: UUID (主キー)  
  - `user_id`: UUID (外部キー、`users`テーブルの`id`を参照、ユーザー削除時にカスケード削除)  
  - `recipe_id`: UUID (外部キー、`recipes`テーブルの`id`を参照、レシピ削除時にカスケード削除)  
  - `viewed_at`: TIMESTAMP (NULL不可, デフォルト CURRENT_TIMESTAMP)  
  - UNIQUE(`user_id`, `recipe_id`, `viewed_at`): 各ユーザーは同じレシピを同じ閲覧時刻に複数回記録できません。

### 3.3 API設計

RESTful API設計原則に従ったエンドポイント設計

- 認証系  
  - POST /api/auth/register - ユーザー登録  
  - POST /api/auth/login - ログイン  
  - POST /api/auth/logout - ログアウト  
  - GET /api/auth/me - 現在のユーザー情報取得  
  - POST /api/auth/password-reset - パスワードリセット  
- ユーザー系  
  - GET /api/users/:id - ユーザー情報取得  
  - PUT /api/users/:id - ユーザー情報更新  
  - GET /api/users/:id/recipes - ユーザーのレシピ一覧  
  - GET /api/users/:id/favorites - ユーザーのお気に入り一覧  
  - POST /api/users/:id/follow - ユーザーをフォロー  
  - DELETE /api/users/:id/follow - フォロー解除  
- レシピ系  
  - GET /api/recipes - レシピ一覧取得  
  - GET /api/recipes/:id - レシピ詳細取得  
  - POST /api/recipes - レシピ作成  
  - PUT /api/recipes/:id - レシピ更新  
  - DELETE /api/recipes/:id - レシピ削除  
  - POST /api/recipes/:id/like - 「おいしい」追加  
  - DELETE /api/recipes/:id/like - 「おいしい」削除  
- 材料系  
  - GET /api/ingredients - 材料一覧取得  
  - GET /api/ingredients/:id - 材料詳細取得  
  - GET /api/users/me/ingredients - ユーザーの手持ち材料取得  
  - POST /api/users/me/ingredients - 手持ち材料追加  
  - DELETE /api/users/me/ingredients/:id - 手持ち材料削除  
- AI系  
  - POST /api/ai/generate-recipe - AIカクテルレシピ生成  
- 閲覧履歴系  
  - GET /api/users/me/browsing-history - ユーザーの閲覧履歴一覧取得  
  - POST /api/recipes/:id/view - レシピ閲覧履歴登録  
- 通知系  
  - GET /api/notifications - 通知一覧取得  
  - PUT /api/notifications/:id/read - 通知を既読にする  
  - PUT /api/notifications/read-all - 全通知を既読にする

- モニタリング  
  - アクセスログ収集・分析  
  - エラー検知・通知  
    - Sentry（エラートラッキング） を使用し、フロントエンドおよびバックエンドのエラーを捕捉し、エラーのグルーピングやアラート設定を行うことで、効率的なエラー管理を実現  
  - パフォーマンス測定  
    - Datadog を使用して、Web Vitals（LCP, FID, CLS, TTFB）やサーバーサイドメトリクス（レスポンスタイム、エラーレート、リクエスト数） の監視を行います  
    - 高エラーレート（5%超過）、高レイテンシ（p95 > 1000ms）、サーバーダウン、異常トラフィックなどの閾値を設定し、異常を検知した際にはアラートを通知します
