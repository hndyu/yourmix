# 1. 設計方針

- 個人開発・低コストを前提とした構成
- Cloudflare エコシステムを最大限活用
- MVP を意識しつつ、将来拡張を阻害しない設計
- フロントエンド主導（BFF 的 API）構成

---

# 2. システム構成

## 2.1 全体アーキテクチャ

- アーキテクチャスタイル：JAMstack
- フロントエンド／バックエンド：Next.js（App Router）

  - UI：React
  - API：Next.js API Routes（BFF）
- API 設計：REST
- フォルダ構成：

  - feature-based を基本
  - feature 内は type-based（components / hooks / services など）

> ※ SSR / SSG / ISR はページ特性に応じて併用

---

## 2.2 技術スタック

| 領域       | 技術                                  |
| -------- | ----------------------------------- |
| UI       | Next.js / React / Tailwind.css                   |
| 状態管理     | React Context / URL State（必要に応じて導入） |
| 認証       | better-auth-cloudflare                        |
| AI       | Gemini API                          |
| DB       | Cloudflare D1 (SQLite)              |
| 画像       | Cloudflare Image Resizing           |
| 分析       | GA4 / Clarity                       |
| 決済       | Stripe（将来）                          |

---

## 2.3 インフラ構成（Cloudflare）

- ホスティング：Cloudflare Workers
- CDN：Cloudflare Edge Network
- データベース：Cloudflare D1
- ストレージ：Cloudflare Static Assets（画像・静的ファイル）

---

# 3. セキュリティ設計

## 3.1 認証・認可

- better-auth-cloudflare を利用した認証
- HTTPOnly Cookie によるセッション管理
- CORS 制御
- API レート制限

## 3.2 データ保護

- HTTPS 強制（TLS 1.3）
- 個人情報の暗号化保存
- XSS / CSRF 対策

# 10. 設計上の補足

- MVP 段階ではすべてを実装しない
- 状態管理・課金・監修者機能は段階導入
- 実装判断は「ユーザー体験への寄与」を最優先とする
