# 基本設計
システム全体の構成と基本的な設計方針を記載する。

## システム構成

### アーキテクチャ概要
- JAMstack アーキテクチャ採用
  - フロントエンド：Next.js（SSG/ISRモード）
  バックエンド：Next.js API Routes（サーバーレス関数）
  - API連携：RESTful API
  - 状態管理：React Context + SWR
  - UIライブラリ：Tailwind CSS

### インフラストラクチャ
- クラウドプラットフォーム：Cloudflare
  - フロントエンドホスティング：Cloudflare Worker
  - データベース：Cloudflare D1（SQLite互換）
  - ストレージ：Cloudflare R2を検討（画像・静的アセット用）
  - CDN：Cloudflareエッジネットワーク

### セキュリティ設計
- 認証・認可
  - JWTベースの認証システム
  - セッション管理（HTTPOnly Cookie）
  - CORS設定
  - レート制限（API呼び出し制限）
- データ保護
  - 個人情報の暗号化保存
  - HTTPS強制（TLS 1.3）
  - XSS/CSRF対策実装
- コンテンツセキュリティ
  - ユーザー生成コンテンツの検証
  - 画像アップロード制限（サイズ、形式）

## データ設計

### エンティティ関連図（ER図）
主要なエンティティとその関連性を定義：
- User（ユーザー）
  - UserProfile（ユーザープロフィール）[1:1]
  - UserIngredient（ユーザー所持材料）[1:n]
  - Recipe（レシピ）[1:n]
  - Like（「おいしい」）[1:n]
  - Follow（フォロー関係）[n:n]
- Notification（通知）
  - User（ユーザー）[n:1]
- Recipe（レシピ）
  - RecipeIngredient（レシピ材料）[1:n]
  - RecipeImage（レシピ画像）[1:n]
  - RecipeTag（レシピタグ）[n:n]
- Ingredient（材料マスタ）
  - IngredientCategory（材料カテゴリ）[n:1]
  - UserIngredient（ユーザー所持材料）[1:n]

### データモデル
各エンティティの属性とデータ型を定義
- User
  - id: UUID（主キー）
  - email: STRING（一意）
  - password_hash: STRING
  - date_of_birth: DATE 
  - created_at: DATETIME
  - updated_at: DATETIME
- UserProfile
  - id: UUID（主キー）
  - user_id: UUID（外部キー）
  - display_name: STRING
  - bio: TEXT
  - avatar_url: STRING
  - preferences: JSON
  - created_at: DATETIME
  - updated_at: DATETIME
- Notification（通知）
  - id: UUID（主キー）
  - user_id: UUID（外部キー, 通知を受け取るUserへの参照）
  - type: ENUM('like', 'follow', 'variant', 'admin_message')（通知の種類）
  - source_id: UUID（関連するエンティティID: レシピID, ユーザーIDなど, NULLABLE）
  - message: TEXT（通知メッセージ, またはタイプとソースIDから動的に生成）
  - is_read: BOOLEAN（既読フラグ, デフォルトはfalse）
  - created_at: DATETIME
- Recipe
  - id: UUID（主キー）
  - user_id: UUID（外部キー）
  - title: STRING
  - description: TEXT
  - prep_time: INTEGER
  - strength: ENUM('non_alcoholic', 'light', 'medium', 'strong')
  - taste_profile: JSON
  - type: ENUM('basic', 'variant', 'ai_generated')
  - parent_recipe_id: UUID（外部キー、派生レシピの場合）
  - view_count: INTEGER
  - created_at: DATETIME
  - updated_at: DATETIME
- RecipeTags
  - id: UUID（主キー）
  - recipe_id: UUID（外部キー）
  - title: STRING
  - created_at: DATETIME
  - updated_at: DATETIME
- Like
  - id: UUID（主キー）
  - user_id: UUID（外部キー）
  - recipe_id: UUID（外部キー）
  - created_at: DATETIME
  - updated_at: DATETIME
- Follow
  - id: UUID（主キー）
  - follower_id: UUID（外部キー）
  - following_id: UUID（外部キー）
  - created_at: DATETIME
  - updated_at: DATETIME
- Ingredient
  - id: UUID（主キー）
  - name: STRING
  - description: TEXT
  - category_id: UUID（外部キー）
  - alcohol_percentage: FLOAT
  - image_url: STRING
  - affiliate_link: STRING
  - created_at: DATETIME
  - updated_at: DATETIME
- UserIngredients
  - id: UUID（主キー）
  - user_id: UUID（外部キー）
  - ingredient_id: UUID（外部キー）
  - quantity: ENUM('plenty', 'some', 'low', 'none')
  - created_at: DATETIME
  - updated_at: DATETIME


### API設計
RESTful API設計原則に従ったエンドポイント設計
- 認証系
  - POST /api/auth/register - ユーザー登録
  - POST /api/auth/login - ログイン
  - POST /api/auth/logout - ログアウト
  - GET /api/auth/me - 現在のユーザー情報取得
  - POST /api/auth/password-reset - パスワードリセット
- ユーザー系
  - GET /api/users/:id - ユーザー情報取得
  - PUT /api/users/:id - ユーザー情報更新
  - GET /api/users/:id/recipes - ユーザーのレシピ一覧
  - GET /api/users/:id/favorites - ユーザーのお気に入り一覧
  - POST /api/users/:id/follow - ユーザーをフォロー
  - DELETE /api/users/:id/follow - フォロー解除
- レシピ系
  - GET /api/recipes - レシピ一覧取得
  - GET /api/recipes/:id - レシピ詳細取得
  - POST /api/recipes - レシピ作成
  - PUT /api/recipes/:id - レシピ更新
  - DELETE /api/recipes/:id - レシピ削除
  - POST /api/recipes/:id/like - 「おいしい」追加
  - DELETE /api/recipes/:id/like - 「おいしい」削除
- 材料系
  - GET /api/ingredients - 材料一覧取得
  - GET /api/ingredients/:id - 材料詳細取得
  - GET /api/users/me/ingredients - ユーザーの手持ち材料取得
  - POST /api/users/me/ingredients - 手持ち材料追加
  - DELETE /api/users/me/ingredients/:id - 手持ち材料削除
- AI系
  - POST /api/ai/generate-recipe - AIカクテルレシピ生成
  - POST /api/ai/save-recipe - 生成レシピ保存
